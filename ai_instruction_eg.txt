## 1.3 Cargo & Build System Mastery

### 1.3.1 Release Profiles

```toml
[profile.release]
opt-level = 3        # Maximum speed optimization
lto = "thin"         # Link-Time Optimization (thin = faster than "fat")
codegen-units = 1    # Single codegen unit = better inlining
strip = "symbols"    # Remove debug symbols → smaller binary
panic = "abort"      # No stack unwinding → smaller binary (good for mobile)
```

> **Hardware Metaphor:** `lto` is like having the linker do a final "assembly optimization pass" across all object files — like a factory foreman reorganizing the entire assembly line after all parts arrive.

> **Mobile Note:** On Termux/Android, binary size matters. `strip = "symbols"` + `panic = "abort"` can reduce binary size by 30–50%.

### 1.3.2 Cargo Features for Platform Targets

```toml
[features]
default = []
termux = []      # Termux-specific code paths
desktop = []     # Desktop-only features (ratatui TUI, etc.)
```

Usage in code:

```rust
#[cfg(feature = "termux")]
fn platform_path() -> &'static str { "/data/data/com.termux/files/home" }

#[cfg(feature = "desktop")]
fn platform_path() -> &'static str { "/home/user" }
```

Build with:

```bash
cargo build --release --features termux
```

### 1.3.3 Cargo.toml vs Cargo.lock

| File | Role | Commit to Git? |
|---|---|---|
| `Cargo.toml` | Your **intent** — version ranges, features | ✅ Always |
| `Cargo.lock` | Exact resolved versions — **reproducibility** | ✅ For binaries; optional for libs |

> **Hardware Metaphor:** `Cargo.toml` is your **bill of materials** (BOM). `Cargo.lock` is the **actual serial numbers** of every component used on the production line.

### 1.3.4 Cross-Compilation to aarch64 (Termux)

```bash
# Option A: Using `cross` (Docker-based, easiest)
cargo install cross
cross build --release --target aarch64-linux-android

# Option B: cargo-ndk (Android NDK required)
cargo install cargo-ndk
cargo ndk -t arm64-v8a build --release
```

> **Note:** For proot-distro Ubuntu on Termux, target may be `aarch64-unknown-linux-gnu` instead of Android.
